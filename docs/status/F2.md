## 2026-07-20

Done (Poulav, per the 07-19 handoff transferring F2 gateway work):
- Added seller-side x402 charging to apps/firm-gateway. Treasury was not copied
  from — it is absent from this repo and untouchable — so this is built from the
  x402 wire format directly, with the unverified parts marked rather than guessed.
  - `src/charging.ts` — challenge construction, `PAYMENT-REQUIRED` encoding,
    facilitator verification, `PAYMENT-RESPONSE` settlement encoding. No key, no
    transfers: charging is a challenge plus a verification, both key-free. The
    Firm's wallet stays in packages/procurer.
  - The payment gate runs in the HTTP handler BEFORE `toolCall`, so an unpaid
    caller cannot reach a single database write.
- DECISION (asked in the 07-18 entry, and by the handoff): unpaid attempts are
  **rejected before every DB write** and recorded on stderr only. Persisting them
  would hand an unauthenticated caller a write primitive; the facilitator keeps
  its own record of failed verifications. Reversible if the demo needs the audit
  trail more than it needs the write protection.
- `CHARGING_MODE` gates paid tools. `enforce` = unreachable without a verified
  payment. Anything else = BYPASS, which logs a warning at startup and on every
  call, and stamps `charging: "BYPASSED"` on every response so no output from a
  bypassed gateway can be mistaken for a paid run.
- `express_run` returns its TODO placeholder BEFORE the charge gate. Charging for
  a placeholder vendor pool would be taking money for nothing.
- Seller config (FIRM_PAYTO_ADDRESS / FIRM_CHARGE_ASSET / FIRM_CHARGE_NETWORK) has
  no defaults and fails closed: an unconfigured gateway returns 503 rather than
  serving. A wrong payTo sends the user's money to a stranger.
- Verification fails closed in every uncertain case, including when
  X402_FACILITATOR_URL is unset. There is no branch that returns ok without a
  facilitator having said so.
